# План устранения замечаний code review для ролевой модели

## Проблема
В ходе code review выявлены замечания по безопасности и функциональности ролевой модели, требующие доработки.

## Замечания из review

### 1. WARNING: SQL injection в getTicketVisibilityFilter
- **Проблема**: Функция возвращает SQL строку с прямой подстановкой agentId
- **Риск**: Потенциальная уязвимость SQL injection
- **Текущее место**: backend/middleware/permissions.js:124

### 2. SUGGESTION: Жестко задан action 'read' для всех разрешений
- **Проблема**: Все разрешения агента мапятся только на action: 'read'
- **Ограничение**: Нет гранулярности действий (create, update, delete)
- **Текущее место**: backend/controllers/authController.js:134-145

### 3. SUGGESTION: Недостаточные права для обычных пользователей
- **Проблема**: Обычным пользователям даются широкие права 'read' на 'all'
- **Риск**: Слишком permissive для non-agent пользователей
- **Текущее место**: backend/controllers/authController.js:146-152

## Цели доработки
1. Устранить уязвимость SQL injection в фильтрации тикетов
2. Расширить маппинг разрешений на действия CASL
3. Определить адекватные права для разных типов пользователей
