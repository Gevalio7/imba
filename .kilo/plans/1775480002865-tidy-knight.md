# План: Доработка функции performSave для обработки эскалации обращения

## Цель
Реализовать корректную логику эскалирования обращения при сохранении тикета.

## ✅ Статус: РЕАЛИЗОВАНО

Все этапы выполнены в файле `src/pages/apps/tickets/edit.vue`:

### 1. Расширение реактивного объекта ticket ✅
Добавлены поля:
- `observerAgentIds: [] as number[]`
- `observerGroupIds: [] as number[]`
- `escalationCount: 0`
- `isEscalated: false`
- `initialExecutorAgentIds: [] as number[]`
- `initialExecutorGroupIds: [] as number[]`

### 2. Обновление fetchTicket ✅
При загрузке тикета сохраняются начальные значения исполнителей и загружаются поля эскалирования.

### 3. Реализация логики эскалирования в performSave ✅
- **Триггер**: статус с `type === "эскалировано"`
- **Валидация**: блокировка если не выбраны исполнители/группы, показ toast
- **Преобразование**:
  - Добавление старых исполнителей в наблюдатели (с дедупликацией)
  - Увеличение счётчика, установка флага
  - **НЕ меняем статус вручную** — переход происходит автоматически через workflow (флоу)
- **Отладка**: console.log с эмодзи 🚀 для каждого шага

### 4. Обновление ticketData ✅
Добавлены поля эскалирования в объект для отправки.

### 5. Локальное обновление после сохранения ✅
После успешного сохранения обновляются локальные данные тикета.

## Технические детали
- Используется `type` статуса, не `name`
- Добавлены console.log с 🚀 для отладки
- Типизация: используется `any` с пометками `// TODO: типизировать`
- Валидация показывает toast при блокировке
