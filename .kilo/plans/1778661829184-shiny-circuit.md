# Plan: Modify SessionManagement Interface for Active Sessions Management

## Goal
Enhance the SessionManagement interface to display online active sessions with logout capabilities, session termination buttons, automatic termination of previous sessions for the same user upon new login, and a bulk action to terminate all sessions.

## Current State Analysis
- Frontend: Vue component at `src/pages/apps/SessionManagement.vue` manages CRUD for session records.
- Backend: Node.js with models, controllers, and routes for session management.
- Sessions have fields: id, username, ipAddress, userAgent, loginTime, lastActivity, isActive, createdAt, updatedAt.
- Existing bulk actions: delete and change status.

## Requirements Breakdown
1. Display only online active sessions (isActive = true).
2. Add "Terminate Session" button for each record to end individual sessions.
3. Record date/time of last action (update lastActivity) when terminating.
4. When a user logs in again, automatically terminate their previous active sessions.
5. Add bulk action "Terminate All Sessions" (button already exists in bulk menu).

## Implementation Plan

### Backend Changes
1. **Modify Session Model** (`backend/models/sessionManagement.js`):
   - Update `update` method to set `lastActivity` to current timestamp when terminating (setting `isActive` to false).
   - Add method to terminate all active sessions for a user.

2. **Modify Session Controller** (`backend/controllers/sessionManagementController.js`):
   - Add endpoint to terminate a session (set isActive=false, update lastActivity).
   - Modify `createSessionManagement` to check for existing active sessions of the user and terminate them before creating new one.
   - Add endpoint for bulk terminate all sessions.

3. **Update Routes** (`backend/routes/sessionManagement.js`):
   - Add routes for terminate session and bulk terminate.

### Frontend Changes
1. **Modify Vue Component** (`src/pages/apps/SessionManagement.vue`):
   - Change default filter to show only active sessions (statusFilter.value = 1).
   - Add "Terminate" button in actions column for each row.
   - Add new bulk action "Terminate All Sessions".
   - Update terminate functions to call new backend endpoints and handle lastActivity update.
   - Remove or disable add button (already done in previous task).

2. **API Calls**:
   - Add `terminateSession` function to call new terminate endpoint.
   - Add `terminateAllSessions` for bulk action.

### Database Considerations
- Ensure `lastActivity` is updated with current timestamp on termination.
- Add logic in create to terminate previous active sessions for the same username.

## Steps to Execute
1. Backend: Update model to handle termination with lastActivity.
2. Backend: Modify controller for new endpoints and auto-terminate on login.
3. Backend: Add routes.
4. Frontend: Add terminate button and bulk action.
5. Frontend: Update filtering to default active only.
6. Test the integration.

## Potential Questions for User
- Should the interface show only active sessions by default, or all with filtering?
- What should happen to terminated sessions - hide them or keep showing?
- Confirm backend logic for auto-terminating previous sessions on new login.</content>
