# План: Функционал быстрых ответов из базы знаний

## Цель
Реализовать возможность привязки статей из базы знаний к очереди и использования их как быстрых ответов при создании/редактировании тикетов.

## Анализ текущей системы

- **Очереди**: имеют поля `templateId` (ссылка на шаблон), `autoResponseTemplate`, `serviceId`
- **База знаний**: статьи имеют `categoryId`, `serviceId`, `title`, `content`, `tags`
- **Тикеты**: поле description - текстовое описание, комментарии в отдельной вкладке

## Уточнения по требованиям

- **Расположение кнопки**: Только во вкладке комментариев (не в основном описании)
- **Добавление контента**: Preview статьи + кнопка "Добавить" (не автовставка)
- **Интерфейс выбора в очереди**: Список с чекбоксами
- **Автоподстановка при создании**: Нет - только ручной выбор

## Этапы реализации

### Этап 1: Расширение модели очереди

**Файл**: `backend/models/queues.js`

Добавить поле для хранения привязанных статей базы знаний:
- `quickAnswerArticleIds` - массив ID статей из базы знаний (INTEGER[])

### Этап 2: API для получения статей с фильтрами

**Файл**: `backend/controllers/knowledgeBaseController.js`

Добавить функцию `getArticlesByFilters(categoryId, serviceId)`:
- Принимает categoryId и/или serviceId
- Возвращает статьи, соответствующие критериям

**Файл**: `backend/routes/knowledge-base.js`

Добавить маршрут:
- `GET /knowledge-base/by-filters?categoryId=X&serviceId=Y`

### Этап 3: UI редактирования очереди

**Файл**: `src/pages/apps/QueuesAddEdit.vue`

Добавить секцию "Быстрые ответы":

1. Поля для фильтрации:
   - `AppSelect` для выбора категории (из types)
   - `AppSelect` для выбора сервиса (из services)

2. Кнопка "Найти статьи" - загружает статьи по выбранным фильтрам

3. **Список найденных статей (VCard)**:
   - Заголовок статьи
   - Preview контента (первые 100 символов)
   - **Checkbox для выбора статьи** (выбрано пользователем)
   - ID выбранной статьи сохраняется в `quickAnswerArticleIds`

4. Отображение выбранных статей:
   - Список с возможностью удалить

### Этап 4: UI тикета - быстрые ответы

**Файл**: `src/pages/apps/tickets/edit.vue`

Добавить функционал во вкладку комментариев:

1. При выборе очереди (`ticket.queueId`):
   - Загрузить статьи из `quickAnswerArticleIds` выбранной очереди

2. Добавить кнопку "Быстрые ответы" (иконка `bx-book`) рядом с полем комментария:
   - Открывает `VDialog` со списком статей
   - Каждая статья показывает заголовок и preview контента
   - Кнопка "Добавить" рядом с каждой статьей
   - При нажатии "Добавить" - контент статьи вставляется в поле нового комментария

**UI компоненты**:
- `VBtn` с иконкой `bx-book` - "Быстрые ответы"
- `VDialog` со списком статей (preview + кнопка "Добавить")
- При нажатии "Добавить" - вставляется контент статьи в поле комментария

### Этап 5: API обновления

**Файл**: `backend/routes/queues.js`

Обновить методы create/update для поддержки нового поля.

## Технические детали

### База данных
```sql
ALTER TABLE queues ADD COLUMN IF NOT EXISTS quick_answer_article_ids INTEGER[];
```

### Фронтенд
- Zustand или composable для хранения списка доступных быстрых ответов
- Watcher за изменениями `ticket.queueId` для загрузки статей

## Приоритеты реализации

1. Backend: расширение модели и API (критично)
2. Frontend: редактирование очереди (для привязки статей)
3. Frontend: тикет с быстрыми ответами (для использования)
