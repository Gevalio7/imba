# Plan: Ролевая модель с read/write/delete разрешениями

## Цель
Реализовать систему управления разрешениями где:
1. API возвращает данные в формате `{ code, is_granted }`
2. Фронтенд преобразует это во внутреннее представление `{ code, read, write, delete }`
3. При сохранении отправляется формат `{ "permissions": { "code_read": true } }`
4. Разрешения `read = true` управляют отображением пунктов в навигационном меню

## Проблема
В текущем коде `src/pages/apps/roles/edit.vue`:
1. Функция `setChildPermission` дублируется (строки 338-391 и 393-456)
2. `fetchPermissions` преобразует `is_granted` но не проверяет правильно ключи в `permissionsMap`
3. При перезагрузке страницы галочки не сохраняются
4. Навигационное меню не использует разрешения (в `casl.ts` проверка отключена)

### Шаг 4: Создать Composable для хранения разрешений пользователя
**Файл:** `src/composables/useUserPermissions.ts` (новый)

```typescript
import { ref, computed } from 'vue'

interface Permission {
  code: string
  read: boolean
  write: boolean
  delete: boolean
}

export function useUserPermissions() {
  const permissionsMap = ref<Map<string, Permission>>(new Map())
  
  // Загрузить разрешения для текущего пользователя
  const loadPermissions = async () => {
    const data = await $api<{ permissions: Permission[] }>('/user/permissions')
    data.permissions.forEach(perm => {
      const type = perm.code.endsWith('_read') ? 'read' : perm.code.endsWith('_write') ? 'write' : 'delete'
      permissionsMap.value.set(perm.code, {
        code: perm.code,
        read: type === 'read' ? perm.read : false,
        write: type === 'write' ? perm.write : false,
        delete: type === 'delete' ? perm.delete : false
      })
    })
  }
  
  // Проверить разрешение на чтение
  const canRead = (code: string): boolean => {
    const perm = permissionsMap.value.get(`${code}_read`)
    return perm?.read === true
  }
  
  // Проверить разрешение на запись
  const canWrite = (code: string): boolean => {
    const perm = permissionsMap.value.get(`${code}_write`)
    return perm?.write === true
  }
  
  // Проверить разрешение на удаление
  const canDelete = (code: string): boolean => {
    const perm = permissionsMap.value.get(`${code}_delete`)
    return perm?.delete === true
  }
  
  return {
    permissionsMap,
    loadPermissions,
    canRead,
    canWrite,
    canDelete
  }
}
```

### Шаг 5: Интегрировать проверку разрешений в навигацию
**Файл:** `src/@layouts/plugins/casl.ts`

Заменить `return true` на вызов `canRead` для пунктов меню:

```typescript
import { useUserPermissions } from '@/composables/useUserPermissions'

export const can = (action: string | undefined, subject: string | undefined) => {
  const { canRead } = useUserPermissions()
  
  if (!action || !subject) return true
  
  // Для навигации проверяем read разрешение
  if (action === 'read') {
    return canRead(subject)
  }
  
  return true
}

export const canViewNavMenuGroup = (item: NavGroup) => {
  const { canRead } = useUserPermissions()
  
  const hasAnyVisibleChild = item.children?.some(child => {
    if (child.action === 'read' && child.subject) {
      return canRead(child.subject)
    }
    return true
  })
  
  if (!(item.action && item.subject))
    return hasAnyVisibleChild ?? true
  
  return canRead(item.subject) && (hasAnyVisibleChild ?? true)
}
```
