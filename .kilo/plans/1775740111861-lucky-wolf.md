# План исправления несоответствия ролей агентов

## Проблема

Пользователь Морозова:
- В таблице БД `agents.role_id = 10` (Администратор)
- В UI показывается "Специалист"

**Причина:** На UI роли отображаются через группы агента (таблица `agents_groups`), а не через прямую связь `agents.role_id`. Морозова состоит в группе "Первая линия поддержки", у которой роль "Специалист" (role_id=11).

## Архитектурная проблема

Есть два источника информации о роли агента:
1. **Прямое поле `agents.role_id`** - исторический/устаревший способ
2. **Роли через группы** - текущий способ (`agents_groups_agents` → `agents_groups.role_id` или `agents_groups_roles`)

Код во frontend (`AgentsTable.vue:454-482`) использует ТОЛЬКО роли из групп, игнорируя `agents.role_id`.

## Решение

### Вариант 1: Удалить устаревшее поле `agents.role_id` (Рекомендуется)

Если роли должны быть только через группы:

1. **Миграция данных**
   - Создать скрипт миграции, который для каждого агента с `role_id`:
     - Проверяет, есть ли у агента группы
     - Если нет групп - создать дефолтную группу с ролью агента
     - Привязать агента к этой группе

2. **Удаление поля из БД**
   - Создать миграцию для удаления колонки `agents.role_id`

3. **Обновление кода**
   - Убрать все упоминания `role_id` из:
     - `backend/models/agents.js` (строки 110, 290-291 в roles.js)
     - Любые другие места, где используется `agents.role_id`

### Вариант 2: Синхронизировать оба источника

Если нужно поддерживать оба способа:

1. **Backend**
   - В `agents.js` метод `getAll()` - собирать роли из ОБОИХ источников:
     - Прямая роль из `agents.role_id`
     - Роли из групп
   - При обновлении агента - синхронизировать `agents.role_id` с основной ролью из групп

2. **Frontend** 
   - В `AgentsTable.vue` метод `getAgentRoles()` - учитывать прямую роль агента:
     ```javascript
     // Добавить проверку item.roleId перед получением ролей из групп
     const directRoleId = item.roleId
     ```

### Вариант 3: Только прямые роли (откат к старой модели)

Если группы не должны определять роли:

1. **Удалить роли из групп**
   - Очистить таблицу `agents_groups_roles`
   - Удалить поле `role_id` из `agents_groups`

2. **Обновить frontend**
   - Изменить `AgentsTable.vue` - отображать роль из `item.roleId`

## Рекомендация

**Вариант 1** - самый чистый подход. Роли через группы - более гибкая система (один агент может иметь несколько ролей через разные группы).

## Детали реализации (Вариант 1)

### Шаг 1: Миграция данных

```javascript
// backend/migrate-agents-roles-to-groups.js
const { pool } = require('./config/db');

async function migrateAgentsRolesToGroups() {
  const client = await pool.connect();
  try {
    await client.query('BEGIN');
    
    // Найти всех агентов с role_id
    const agentsWithRoles = await client.query(`
      SELECT id, role_id, first_name, last_name 
      FROM agents 
      WHERE role_id IS NOT NULL
    `);
    
    for (const agent of agentsWithRoles.rows) {
      // Проверить, есть ли у агента группы
      const groupsCheck = await client.query(`
        SELECT COUNT(*) as count 
        FROM agents_groups_agents 
        WHERE agent_id = $1
      `, [agent.id]);
      
      if (groupsCheck.rows[0].count === '0') {
        // У агента нет групп - создать дефолтную группу для его роли
        const roleName = await client.query(`
          SELECT name FROM roles WHERE id = $1
        `, [agent.role_id]);
        
        if (roleName.rows.length > 0) {
          const groupName = `Группа ${roleName.rows[0].name}`;
          
          // Проверить, есть ли уже такая группа
          let group = await client.query(`
            SELECT id FROM agents_groups 
            WHERE name = $1 AND role_id = $2
          `, [groupName, agent.role_id]);
          
          if (group.rows.length === 0) {
            // Создать группу
            group = await client.query(`
              INSERT INTO agents_groups (name, role_id, is_active) 
              VALUES ($1, $2, true) 
              RETURNING id
            `, [groupName, agent.role_id]);
          }
          
          // Добавить агента в группу
          await client.query(`
            INSERT INTO agents_groups_agents (agents_group_id, agent_id)
            VALUES ($1, $2)
            ON CONFLICT DO NOTHING
          `, [group.rows[0].id, agent.id]);
          
          // Добавить роль в agents_groups_roles
          await client.query(`
            INSERT INTO agents_groups_roles (agents_group_id, role_id)
            VALUES ($1, $2)
            ON CONFLICT DO NOTHING
          `, [group.rows[0].id, agent.role_id]);
          
          console.log(`✅ Агент ${agent.first_name} ${agent.last_name} (ID ${agent.id}) добавлен в группу "${groupName}"`);
        }
      } else {
        // У агента есть группы - проверить соответствие ролей
        const groupRoles = await client.query(`
          SELECT DISTINCT ag.role_id, r.name
          FROM agents_groups_agents aga
          JOIN agents_groups ag ON aga.agents_group_id = ag.id
          LEFT JOIN roles r ON ag.role_id = r.id
          WHERE aga.agent_id = $1
        `, [agent.id]);
        
        const hasMatchingRole = groupRoles.rows.some(gr => gr.role_id === agent.role_id);
        
        if (!hasMatchingRole) {
          console.log(`⚠️  Агент ${agent.first_name} ${agent.last_name} (ID ${agent.id}): role_id=${agent.role_id} не совпадает с ролями групп`);
          console.log(`   Роли в группах: ${groupRoles.rows.map(r => `${r.role_id}:${r.name}`).join(', ')}`);
        }
      }
    }
    
    await client.query('COMMIT');
    console.log('✅ Миграция завершена');
  } catch (error) {
    await client.query('ROLLBACK');
    console.error('❌ Ошибка миграции:', error);
    throw error;
  } finally {
    client.release();
  }
}

migrateAgentsRolesToGroups().then(() => process.exit(0)).catch(() => process.exit(1));
```
