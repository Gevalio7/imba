# План: Исправление справочника очередей

## Проблема
1. Не все введённые данные сохраняются
2. В табличной части вместо наименований выводятся ID

## Причины проблем

### 1. Backend: Модели не поддерживают фильтр `isActive`

Следующие модели принимают параметр `isActive` от reference-data.js, но не используют его:
- `backend/models/customers.js` - getAll() игнорирует isActive
- `backend/models/customersGroups.js` - getAll() не принимает isActive
- `backend/models/agentsGroups.js` - getAll() не принимает isActive
- `backend/models/templates.js` - getAll() не принимает isActive

### 2. Backend: PostgreSQL массивы не парсятся в JavaScript массивы

В `backend/models/queues.js` поля `executorGroupIds`, `executorAgentIds`, `observerAgentIds`, `quickAnswerArticleIds` возвращаются как строки PostgreSQL (`{1,2,3}`) вместо JavaScript массивов (`[1,2,3]`).

### 3. Backend: reference-data.js не фильтрует данные по isActive

В `backend/routes/reference-data.js`:
- `customersGroups` вызывается без `isActive: true`
- `agentGroups` вызывается без `isActive: true`
- `templates` вызывается без `isActive: true`

## Этапы исправления

### Этап 1: Исправить model customers.js
Добавить поддержку фильтра `isActive` в метод `getAll()`:
- Принимать опцию `isActive` из параметров
- Добавлять условие `WHERE is_active = true` при фильтрации

### Этап 2: Исправить model customersGroups.js
Добавить поддержку фильтра `isActive` в метод `getAll()`:
- Принимать опцию `isActive` из параметров
- Добавлять условие `WHERE is_active = true` при фильтрации

### Этап 3: Исправить model agentsGroups.js
Добавить поддержку фильтра `isActive` в метод `getAll()`:
- Принимать опцию `isActive` из параметров
- Добавлять условие `WHERE is_active = true` при фильтрации

### Этап 4: Исправить model templates.js
Добавить поддержку фильтра `isActive` в метод `getAll()`:
- Принимать опцию `isActive` из параметров
- Добавлять условие `WHERE is_active = true` при фильтрации

### Этап 5: Исправить model queues.js
Добавить парсинг PostgreSQL массивов в JavaScript массивы:
- После получения данных из БД, преобразовать поля `{1,2,3}` в `[1,2,3]`
- Использовать регулярное выражение для извлечения чисел из строки

### Этап 6: Исправить reference-data.js
Обновить вызовы моделей:
- `CustomersGroups.getAll({ itemsPerPage: 1000, isActive: true })`
- `AgentsGroups.getAll({ itemsPerPage: 1000, isActive: true })`
- `Templates.getAll({ itemsPerPage: 1000, isActive: true })`

## Файлы для изменения

1. `backend/models/customers.js` - добавить фильтр isActive

## Альтернативное решение (если проще)

Вместо изменения каждой модели, можно добавить парсинг PostgreSQL массивов прямо в `reference-data.js` при получении данных очередей. Но это менее элегантно.
