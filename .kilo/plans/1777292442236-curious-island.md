# План доработки ролевой модели для видимости элементов и прав доступа

## Проблема
Пользователь под учеткой admin@demo.com получает ошибку 401 "не авторизован" при попытке редактирования справочников. Предполагается проблема с ролевой моделью, где агенты видят только доступные им элементы: настройки и справочники доступны только админам, сотрудники видят только свои тикеты и могут создавать новые.

## Текущая ситуация
- Система RBAC реализована с ролями, группами агентов и разрешениями
- 401 ошибки возникают только при проблемах аутентификации (токен)
- Видимость справочников не фильтруется по ролям в UI
- Тикеты видны всем аутентифицированным пользователям без фильтрации
- Есть неиспользуемая функция getTicketVisibilityFilter

## Цели доработки
1. Фильтрация видимости элементов UI по ролям (справочники только для админов)
2. Видимость тикетов только своих для сотрудников
3. Правильная обработка ошибок доступа (403 вместо 401)
4. Интеграция существующего фильтра тикетов
