# Аудит и план исправлений для persistent cron jobs

## Обнаруженные проблемы

### 1. Баг в scheduler.js:133 - undefined переменная
```javascript
results.push({ success: true, ticketId: newTicket.id, ticketNumber });
```
`ticketNumber` не определена - должна быть `newTicket.ticketNumber`

### 2. Баг в ticketSchedulesController.js:240 - undefined переменная
```javascript
message: `Создан тикет #${ticketNumber}`
```
`ticketNumber` не определена - должна быть `newTicket.ticketNumber`

### 3. Несоответствие в контроллере runSchedule
При клонировании тикета с `schedule.ticketId`:
- scheduler.js добавляет суффикс "(расписание)" к заголовку (строка 63)
- controller НЕ добавляет этот суффикс (строка 198)

### 4. Проблема с ticketId в scheduler
Согласно описанию, иногда `schedule.ticketId` приходит как null/undefined в планировщике. Нужно добавить дополнительное логирование для отладки.

## План исправлений

### Шаг 1: Исправить баг с undefined ticketNumber в scheduler.js:133
```javascript
results.push({ success: true, ticketId: newTicket.id, ticketNumber: newTicket.ticketNumber });
```

### Шаг 2: Исправить баг с undefined ticketNumber в controller.js:240
```javascript
message: `Создан тикет #${newTicket.ticketNumber}`
```

### Шаг 3: Добавить суффикс "(расписание)" в controller runSchedule
При клонировании из ticketId добавить суффикс к title:
```javascript
title: `${originalTicket.title} (расписание)`,
```

### Шаг 4: Добавить отладочное логирование в scheduler
Для диагностики проблемы с null ticketId:
```javascript
console.log(`🔍 SCHEDULER: schedule ID ${schedule.id}, ticketId = ${schedule.ticketId}, title = ${schedule.title}`);
```

### Шаг 5: Протестировать функциональность
- Создать расписание с привязанным тикетом
- Дождаться выполнения (или запустить вручную)
- Проверить логи и созданный тикет
