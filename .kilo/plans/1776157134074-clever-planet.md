# План: Исправление справочника очередей

## Проблемы
1. **ID вместо названий** - в таблице очередей отображаются ID вместо названий для полей: организация, подразделение, группы исполнителей, исполнители, наблюдатели
2. **Данные не сохраняются** - новые поля (executorGroupIds, executorAgentIds, observerAgentIds, companyId, departmentId) не сохраняются корректно

## Анализ

### Проблема 1: ID вместо названий
- В `Queues.vue` уже есть шаблоны для отображения названий (строки 780-869)
- `referenceData` загружается через `fetchReferenceData()`
- **Причина**: Модели справочников `customers.js` и `customersGroups.js` НЕ поддерживают фильтр `isActive`, который передается из reference-data.js
  - `Customers.getAll({ itemsPerPage: 1000, isActive: true })` - isActive не обрабатывается моделью
  - `CustomersGroups.getAll({ itemsPerPage: 1000 })` - isActive даже не передается

### Проблема 2: Данные не сохраняются
- В `QueuesAddEdit.vue` функция `saveQueue` отправляет данные
- В контроллере поля принимаются и передаются в модель
- **Причина**: в модели `queues.js` массивы преобразуются в PostgreSQL синтаксис некорректно - `{1,2,3}` вместо `'{1,2,3}'` или `ARRAY[1,2,3]`

## Этапы исправления

### Этап 1: Исправить модель customers.js
Добавить поддержку фильтра `isActive` в `getAll()`:
- Принять параметр `isActive` из options
- Добавить WHERE условие для `is_active`

### Этап 2: Исправить модель customersGroups.js
Добавить поддержку фильтра `isActive` в `getAll()`:
- Принять параметр `isActive` из options
- Добавить WHERE условие для `is_active`

### Этап 3: Исправить reference-data.js
- `CustomersGroups.getAll({ itemsPerPage: 1000, isActive: true })` - передать isActive=true

### Этап 4: Исправить сохранение массивов в queues.js
В функциях `create()` и `update()` исправить преобразование массивов:
- Текущий код: `{${queue[field].join(',')}}`  
- Нужно: `'${queue[field].join(',')}'` (кавычки для text массива) ИЛИ использовать SQL функцию `ARRAY[...]`

### Этап 5: Проверить frontend
В `src/pages/apps/Queues.vue`:
- Проверить что `fetchReferenceData` загружает все справочники корректно
- Проверить что шаблоны для отображения работают
