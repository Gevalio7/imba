# План: Добавление полей в справочник очередей

## Цель
Добавить в форму создания/редактирования очереди новые поля:
- Выбор организации (company)
- Выбор подразделения (department) - группа клиентов
- Наблюдатели (observers) - выбор из агентов
- Группы агентов (executor groups) - множественный выбор
- Агенты (executors) - множественный выбор

## Этапы

### Этап 1: Backend - Модель Queues
Добавить поля в модель `backend/models/queues.js`:
- `executorGroupIds: number[]` - массив ID групп агентов
- `executorAgentIds: number[]` - массив ID агентов  
- `observerAgentIds: number[]` - массив ID наблюдателей
- `departmentId: number | null` - подразделение (опционально)

### Этап 2: Backend - Справочники
Модифицировать `backend/routes/referenceData.js`:
- Добавить `customers` (организации) - только активные (isActive=true)
- Добавить `customersGroups` (подразделения) - только активные с фильтром по companyId
- Добавить `agents` (агенты) - только активные (isActive=true)
- Фильтрация по isActive для существующих справочников

### Этап 3: Backend - Контроллер
Обновить `backend/controllers/queuesController.js`:
- При GET: возвращать новые поля
- При POST/PUT: принимать и сохранять новые поля

### Этап 4: Frontend - Типы и интерфейсы
Обновить `src/pages/apps/QueuesAddEdit.vue`:
- Добавить поля в интерфейс Queue
- Добавить поля в объект queue
- Добавить computed для справочников (companies, departments, agents, agentGroups)
- Фильтрация: только активные значения

### Этап 5: Frontend - UI
Добавить поля в форму:
- AppSelect для организации (company)
- AppSelect для подразделения (department) - с фильтрацией по company
- AppSelect для наблюдателей (множественный выбор)
- AppSelect для групп агентов (множественный выбор, если allow_multiple_executor_groups=true)
- AppSelect для агентов (множественный выбор, если agent_auto_assign_as_executor=true)

### Этап 6: Логика множественного выбора
Добавить проверку настроек из system_configuration:
- `allow_multiple_executor_groups` - множественный выбор групп агентов
- `allow_multiple_executors` - множественный выбор агентов-исполнителей
- `agent_auto_assign_as_executor` - при создании тикета агент автоматически становится исполнителем

Логика:
- Если allow_multiple_executor_groups=true → множественный выбор групп
- Если agent_auto_assign_as_executor=true → множественный выбор агентов

## Файлы для изменения
1. `backend/models/queues.js` - модель с новыми полями
2. `backend/routes/referenceData.js` - добавление справочников с фильтрацией
3. `backend/controllers/queuesController.js` - обработка новых полей
4. `src/pages/apps/QueuesAddEdit.vue` - форма с новыми полями
