# План: Внедрение persistent job queue для расписаний тикетов

## Текущая проблема
В `backend/server.js:111` используется `setInterval` - in-memory scheduler, который теряет все задачи при перезапуске приложения.

## Решение
Использовать **pg-boss** - persistent job queue на PostgreSQL (уже используется в проекте).

## Реализация

### 1. Установка зависимости
Добавить `pg-boss` в `backend/package.json`

### 2. Интеграция с backend
- Создать модуль `backend/queue/scheduler.js` для инициализации pg-boss
- Заменить `setInterval` на `boss.on('schedule:processDueSchedules')` 
- Интегрировать в `server.js` с graceful shutdown

### 3. API расширение (опционально)
- Эндпоинт для мониторинга состояния очереди: `GET /api/scheduler/status`

### 4. Docker
Добавить healthcheck для queue в docker-compose

## Приоритет
- **Medium** - реализация в рамках существующего PostgreSQL без новых сервисов
