# План: Автозаполнение полей тикета из выбранной очереди

## Цель
При выборе очереди в тикете (как при создании, так и при редактировании) все поля тикета должны автоматически заполняться из данных выбранной очереди.

## Текущее состояние
- `add.vue` (создание тикета) - использует собственную логику, НЕ использует useTicketForm
- `edit.vue` (редактирование тикета через useTicketForm) - использует useTicketForm composable

## Реализовано
✅ Добавлен watcher для автозаполнения полей в `add.vue` и `useTicketForm.ts`

Автозаполнение следующих полей:
1. `companyId` - из `queue.companyId`
2. `serviceId` - из `queue.serviceId`
3. `slaId` - из `queue.slaId`
4. `priorityId` - из `queue.priorityId`
5. `executorGroupIds` - из `queue.agentGroupId` (если не выбраны исполнители)
6. `typeId` - тип с подходящим `workflowId` (если у очереди есть workflow)
7. `categoryId` - из `queue.categoryId`

## Логика
- При выборе очереди ВСЕ поля перезаписываются данными из очереди
- Пропускается только начальная загрузка (чтобы не перезаписать уже загруженные данные)
- Для typeId: ищется тип с подходящим workflowId
