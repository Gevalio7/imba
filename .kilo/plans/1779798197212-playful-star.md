## ДОПОЛНЕНИЕ К ПЛАНУ: Уточнения по результатам код-ревью и текущему состоянию

### Статус реализации после выполнения заданий ИИ-агентом

На основе выполненной работы и полученного код-ревью, в план вносятся следующие уточнения:

---

### 1. MailFetcherService — текущее состояние

**Что реализовано:**
- ✅ Переход на `queue-driven` архитектуру (метод `fetchForActiveQueues`)
- ✅ Per-queue интервалы (поддержка `mailFetchInterval` в очереди)
- ✅ Определение ответов (reply detection) через `findRelatedTicket`
- ✅ Создание комментариев вместо новых тикетов при ответе
- ✅ Отправка приветственных писем (welcome email) через `EmailSenderService`
- ✅ Использование глобальных настроек как fallback
- ✅ Авто-создание Customer User с reset token

**Проблемы, выявленные ревью (требуют исправления):**
| Проблема | Статус | Действие |
|----------|--------|----------|
| `continue` вне цикла (синтаксическая ошибка) | ✅ **ИСПРАВЛЕНО** | Удалён дублированный блок кода |
| Пароль в открытом виде в письме | ✅ **ИСПРАВЛЕНО** | Используется resetLink вместо пароля |
| Ошибка создания комментария — письмо не помечается Seen | ✅ **ИСПРАВЛЕНО** | Добавлен `finally` с markSeen (строка 247-249) |
| Медленные запросы `SELECT *` | ✅ **ОПТИМИЗИРОВАНО** | `findByExternalId` возвращает только нужные поля |
| Сомнительный fallback `getById` | ✅ **ИСПРАВЛЕНО** | Удалён fallback, используется только `getByNumber` |
| Примитивная замена шаблонов | 💡 **ОПЦИОНАЛЬНО** | Вынести в общий template engine |
