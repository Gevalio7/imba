# План: Использование реальных данных через JWT без моков

## Проблема
После рефакторинга пропали реальные данные, появились моки. MSW (Mock Service Worker) включен (`VITE_USE_MSW=true`), но ticket pages используют `$fetch` без JWT токена.

## Анализ текущего состояния

### ✅ Что работает корректно
- MSW включен в режиме 'bypass' - незамоканные запросы идут на реальный API
- Backend работает с JWT авторизацией (middleware `protect`)
- Есть `$api` в `src/utils/api.ts` с автоматическим добавлением JWT токена
- `src/pages/apps/tickets/index.vue` уже использует `$api` корректно

### ❌ Масштаб проблемы

## План реализации (Вариант A - Фаза 1)

### Этап 1: add.vue
1. Заменить `import { $fetch } from 'ofetch'` → `import { $api } from '@/utils/api'`
2. Заменить все вызовы `$fetch(` → `$api(`
3. Убрать `${API_BASE}` из URL т.к. $api уже использует baseURL
   - Было: `await $fetch(\`${API_BASE}/tickets\`, ...)`
   - Стало: `await $api('/tickets', ...)`

### Этап 2: edit.vue
1. Аналогично add.vue

### Этап 3: Schedules.vue
1. Аналогично add.vue

### Этап 4: useReferenceData.ts
1. Заменить `import { $fetch } from 'ofetch'` → `import { $api } from '@/utils/api'`
2. Изменить строку 58: `await $fetch<ReferenceData>(url)` → `await $api<ReferenceData>(url)`
3. Убрать `${API_BASE}` из URL:
   - Было: `\`${API_BASE}/referenceData\``
   - Стало: `'/referenceData'`

## Важные замечания

1. **$api автоматически добавляет baseURL**, поэтому URL должны быть без `${API_BASE}`:
   - Неправильно: `await $api(\`${API_BASE}/tickets\`)`
   - Правильно: `await $api('/tickets')`

2. **MSW остается включенным** - режим 'bypass' пропускает незамоканные запросы

3. **Токен берется из cookie** - `useCookie('accessToken').value` в `src/utils/api.ts`
