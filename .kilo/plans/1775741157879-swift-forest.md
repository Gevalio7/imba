# План оптимизации ticketsController.js

## Цель
Оптимизировать скорость добавления и редактирования тикетов, убрать лишние console.log, упростить код.

## Проблемы текущего кода
1. Много `console.log` (строки 169, 181, 210, 222, 271, 296, 352, 356, 480, 483, 518, 521)
2. В `updateTicket` (строки 556-591) - последовательная загрузка справочников через `Promise.all`, но с накоплением через `neededLookups` - сложно и медленно
3. В `createTicket` - много последовательных запросов (SystemConfiguration, Types, WorkflowTransitions, Agents, SLA)
4. history recording в updateTicket (строки 593-714) делает отдельный INSERT на каждое изменённое поле

## Оптимизации

### 1. Убрать все console.log
- Удалить все отладочные выводы: строки 169, 181, 210, 222, 271, 296, 352, 356, 480, 483, 518, 521

### 2. Оптимизировать createTicket
- Кешировать SystemConfiguration.getAll() - делать 1 раз в начале, а не 2-3 раза
- Использовать Promise.all для параллельных запросов (Types, WorkflowTransitions, Agents)

### 3. Оптимизировать updateTicket
- Сделать параллельную загрузку справочников вместо последовательного накопления
- Объединить TicketHistory.create() в batch insert (1 запрос вместо N)
- Использовать Promise.all для записи в историю

### 4. Изменить возвращаемый результат
- В createTicket не делать二次 запрос к SLA, а рассчитать дедлайны до INSERT
- В updateTicket возвращать updatedTicket без дополнительных выборок

## Реализация

### Кеширование конфигурации
```javascript
let configCache = null;
async function getConfig(key) {
  if (!configCache) {
    const result = await SystemConfiguration.getAll({ itemsPerPage: 50 });
    configCache = result.systemConfiguration || [];
  }
  return configCache.find(c => c.key === key);
}
```
