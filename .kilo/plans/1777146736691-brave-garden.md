# Plan to Fix SES Uncaught Exception in lockdown-install.js

## Problem Description
The application is showing repeated errors:
```
lockdown-install.js:1 SES_UNCAUGHT_EXCEPTION: null
```
This occurs after adding an observer groups field to the queues form, which references the agent groups directory. The errors suggest that the Secure EcmaScript (SES) shim is removing unpermitted intrinsics, likely related to MSW (Mock Service Worker) usage.

## Observations from Logs
- MSW is initialized and running in bypass mode
- The application fetches queues data successfully before the errors start
- The errors appear to be triggered after certain component interactions (QueuesAddEdit.vue)
- VITE_USE_MSW is undefined, but MSW is still active

## Possible Causes
1. MSW's service worker implementation may be using intrinsics not permitted by the SES shim
2. The SES shim might be overly restrictive in this environment
3. There could be a conflict between MSW and Vue's reactivity system when handling the new observer groups field
4. The error might be related to how MSW handles requests when in bypass mode but still initializing

## Plan of Action

### 1. Investigate MSW and SES Integration
- Check if lockdown-install.js is part of MSW or a separate SES shim
- Determine if we can configure SES to allow required intrinsics for MSW
- Look for similar issues in MSW/Vue/Vite projects

### 2. Examine Recent Changes
- Review the changes made to QueuesAddEdit.vue for the observer groups field
- Check how the groups are fetched and used in the component
- Verify if there are any circular dependencies or infinite loops introduced

### 3. Test MSW Configuration
- Try disabling MSW temporarily to see if errors persist
- Experiment with MSW modes (bypass vs mocked)
- Check if the issue occurs only in development or also in production build

### 4. Consider Alternative Approaches
- If MSW is not essential for development, consider removing it temporarily
- Look into updating MSW to a version that might be more compatible with SES
- Explore configuring the SES shim if it's being used explicitly in the project

### 5. Specific Fixes to Try
- Add necessary intrinsics to SES allowed list if we can identify which ones are missing
- Check if the error is related to specific browser APIs being accessed by MSW
- Verify that the new observer groups field doesn't cause excessive re-renders or infinite update loops
