# План исправления ситуации с гранулярными правами доступа

## Текущая ситуация
1. В БД есть разрешения, которых нет в API (например, `menu_chat_read`, `menu_agents_write`, и т.д.)
2. В API есть разрешения, которых нет в БД (например, `see_company_tickets`, `manage_roles`, и т.д.)
3. Метод `getAvailablePermissions` в модели `Roles` возвращает только 25 базовых разрешений
4. У роли "Специалист" (ID=11) нет необходимых разрешений для управления ролями
5. В интерфейсе редактирования ролей отображаются не все разрешения, доступные в БД

## Цель
Реализовать полноценную систему гранулярных прав доступа, которая позволяет:
1. Отображать все разрешения из БД в интерфейсе редактирования ролей
2. Корректно сохранять и обновлять разрешения
3. Предоставить пользователю Morozova возможность настраивать доступ через интерфейс

## План действий

### 1. Исправление модели Roles (backend/models/roles.js)
- Обновить метод `getAvailablePermissions()` для включения всех разрешений из БД
- Добавить недостающие разрешения для чата и агентов
- Убедиться, что все разрешения из БД представлены в этом списке

### 2. Исправление API endpoint (backend/controllers/rolesController.js)
- Модифицировать метод `getAvailablePermissions` для возврата всех разрешений из БД
- Добавить импорт `pool` для доступа к БД
- Модифицировать метод для получения всех разрешений, а не только базовых

### 3. Обновление разрешений для роли "Специалист" (ID=11)
- Добавить разрешение `manage_roles` для роли "Специалист"
- Убедиться, что роль имеет доступ к управлению разрешениями

### 4. Исправление интерфейса редактирования ролей (src/pages/apps/roles/edit.vue)
- Обновить загрузку разрешений для отображения всех доступных разрешений
- Исправить отображение разрешений, которые есть в БД, но отсутствуют в API
- Проверить корректность работы методов `setChildPermission` и `toggleCategory`

### 5. Тестирование и проверка
- Перезапустить сервер после внесения изменений
- Проверить, что `/roles/permissions` возвращает все разрешения
- Убедиться, что в интерфейсе `edit.vue` отображаются все разрешения
- Протестировать сохранение изменений разрешений через интерфейс
